## 一、现状摘要（已确认）

- 已具备市场数据自动更新脚本（FRED + Barchart + 回退链路）。
- 已具备新闻与学术抓取、微信手动补录面板。
- 已具备 GitHub Actions 自动更新和部署。

## 二、核心问题（当前最影响效果）

1. 数据质量风险
- 学术抓取结果明显跑偏，出现与 LNG 无关条目和异常年份（如 2121）。

2. 本地与线上流程不一致
- CI 在部署前执行 update:data，但本地 docs:build 未串联 update:data，容易出现“本地预览数据旧、线上数据新”。

3. 包管理策略不一致
- 说明文档偏向 npm，CI 使用 pnpm，容易导致团队操作分叉。

4. SEO 与双语联动未体系化
- 已有中英文站点结构，但缺少系统化 SEO 基线（hreflang、结构化数据、统一元信息策略）。

5. 文档职责边界不清
- aiwork 下 project/memory/datasource-plan 存在“计划、事实、日志”交叉，后续 AI 与人工都容易误读。

## 三、优先级路线图

## P0（1-3 天，先做）

### 任务 P0-1：修复学术数据质量
- 动作：
  - 收紧 Crossref 查询条件（限定 subject/container-title 关键词）。
  - 增加结果过滤规则：标题包含 LNG 相关关键词、发布日期范围合理、去重。
  - 增加“抓取为空时回退源”日志标识与计数。
- 验收标准：
  - news-digest academic 前 10 条中，LNG/天然气相关性 >= 80%。
  - 不再出现明显异常未来年份。
- 预估工时：0.5-1 天。

### 任务 P0-2：统一构建入口
- 动作：
  - 将 docs:build 串联 update:data（例如先更新再构建）。
  - 保留独立 update:data 便于手动执行。
- 验收标准：
  - 本地与 CI 的构建前置动作一致。
  - 新人按 README 一次执行即可得到最新数据页面。
- 预估工时：0.5 天。

### 任务 P0-3：建立数据健康摘要
- 动作：
  - 在数据更新脚本末尾新增 health JSON（源命中、回退次数、空值、最新时间戳）。
  - 在市场页展示“更新时间 + 数据健康状态（正常/降级）”。
- 验收标准：
  - 每次 update:data 产出健康文件。
  - 页面可看见当前是否处于回退模式。
- 预估工时：1 天。

## P1（1-2 周）

### 任务 P1-1：SEO 基线建设
- 动作：
  - 为核心栏目页补齐 title/description 模板。
  - 增加中英文互链策略（hreflang 规则）。
  - 为报告/文章页增加结构化数据（Article/Breadcrumb）。
- 验收标准：
  - 核心页面可通过基础 SEO 检查。
  - 中英文对应页面具备互相指向能力。
- 预估工时：2-3 天。

### 任务 P1-2：双语内容一致性检查
- 动作：
  - 新增脚本扫描中文与英文目录差异，生成缺失清单。
  - 设定优先翻译队列（高流量页面优先）。
- 验收标准：
  - 每周可自动输出中英文缺失页面列表。
- 预估工时：1 天。

### 任务 P1-3：aiwork 文档治理
- 动作：
  - project.md 仅保留“当前事实与配置”。
  - datasource-plan.md 仅保留“方案与路线图”。
  - memory.md 仅保留“时间序列日志”。
- 验收标准：
  - 三份文档职责清晰，无交叉重复段落。
- 预估工时：0.5 天。

## P2（2-4 周）

### 任务 P2-1：内容生产流水线
- 动作：
  - 统一文章模板（摘要、关键词、图表、结论、英文同步状态）。
  - 制定发布节奏（例如每日快讯 + 每周深度）。
- 验收标准：
  - 连续 2 周按节奏发布，无明显断档。
- 预估工时：2-3 天（含流程试运行）。

### 任务 P2-2：终端行情页产品化
- 动作：
  - 增加来源说明、回退说明、异常提示。
  - 增加时间维度切换（近 7 天 / 30 天）与对比视图。
- 验收标准：
  - 页面可用于快速决策，不只是静态展示。
- 预估工时：3-5 天。

## 四、建议执行顺序（最小可行路径）

1. P0-1 学术数据质量修复
2. P0-2 构建入口统一
3. P0-3 数据健康摘要
4. P1-1 SEO 基线
5. P1-2 双语一致性脚本
6. P1-3 aiwork 文档治理

## 五、每周检查项（建议固定）

- 市场数据是否出现连续回退。
- 新闻与学术条目相关性是否下降。
- 中英文目录差异是否扩大。
- 本周新增页面是否具备基础 SEO 元信息。
