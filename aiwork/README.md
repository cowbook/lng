# aiwork
告诉AI如何完善本项目, 以及工作原则, 项目维护者叫mark

## 必须要做的事情
在提交到GitHub库之前, 完善本目录如下文件,各个文件的作用如下:

### project.md
这保持着本项目的基本的信息,如项目名称框架、技术、配置环境、用到数据源的URL基本情况等

### memory.md
按时间顺序记录项目完善的情况


### optimization-plan.md
这个文件保持着最重要需要完善的任务, 如果任务没有完成,在提交到GitHub库之前要提醒mark, 要及时删除已经完成的任务, 保持内容清洁.


### datasource-plan.md
本项目需要的数据源, 以及准备完善的计划在这个文件保存.
